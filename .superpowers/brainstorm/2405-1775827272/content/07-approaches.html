3 Approaches to Build This

Same vision, different tech stacks. Each has trade-offs for your showstopper animation goals and Netlify deployment.

A

Pure HTML/CSS/JS + Three.js + GSAP

Recommended. Single static site — no framework overhead. Three.js for 3D coffee elements, GSAP + ScrollTrigger for cinematic scroll animations, particle effects via canvas. Deploys instantly to Netlify as static files. Maximum creative control, fastest load times, zero build step complexity.

Pros: Full animation control, instant Netlify deploy, no dependencies to break, lightweight
Cons: No component reuse (fine for 5 pages), manual routing
B

Astro + Three.js + GSAP

Static site generator with component architecture. Same animation libraries but with proper templating, shared layouts, and markdown blog support. Builds to static HTML for Netlify. Good if blog content will grow.

Pros: Component reuse, built-in blog/markdown, great DX
Cons: Build step required, framework learning curve, slightly more complex deploy
C

Next.js + React Three Fiber + Framer Motion

Full React framework. React Three Fiber for 3D, Framer Motion for UI animations. Most "modern" stack but heaviest. Requires Netlify adapter or static export.

Pros: Largest ecosystem, easy to add e-commerce later, component-driven
Cons: Heaviest bundle, slower initial load, overkill for 5 static pages, more complex Netlify config

My Recommendation: Option A

For a 5-page showstopper site deploying to Netlify, pure HTML/CSS/JS with Three.js and GSAP gives you maximum animation control with zero framework overhead. The site is a visual experience first — a framework would just add weight without benefit. You can always migrate to Astro later if the blog grows.
